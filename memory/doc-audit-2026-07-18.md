## Findings (ranked)

### F1 — CLAUDE.md is 73% rolling changelog (HIGH)
"Current State" is 57KB of the 78KB file; the single "Last Updated" bullet is a 27KB
reverse-chronological narrative of every PB/SR since May. This violates the file's own header
rule ("Worker sessions: append detail there, not here. CLAUDE.md tracks current snapshot
only") and costs ~20k tokens of context in **every** session, coordinator and worker alike.
The history it carries already exists in `memory/workstream-state.md` handoffs, the ESM task
record, and git log.

### F2 — Five docs are stale but presented as current (HIGH)
Ranked by risk of misleading a fresh session:
1. **docs/project-status.md** — self-declares "single source of truth"; reports 2,648 tests
   (real: 3,476), 194/204 abilities, retired W1–W6 model, "next: PB-D". No date banner.
2. **docs/workstream-coordination.md** — "Load this file at the start of any session"; frozen
   at 2026-03-08, describes the retired workstream system. CLAUDE.md's routing table still
   sends "deciding what to work on" here.
3. **docs/primitive-card-plan.md** — literal "## Current State" heading holding March data
   (814/1,452 = 56% TODOs, PB-23..37 as the plan).
4. **docs/dsl-gap-closure-plan.md** — same 56%-TODO framing as the active plan; superseded by
   EF/OS queues.
5. **docs/mtg-engine-low-issues-remediation.md** — in the Primary Documents table; claims
   "~68 OPEN LOW, 1,033 tests"; ~6 LOW actually remain.

Honorable mentions: `card-authoring-operations.md` and `ability-batch-plan.md` (completed
campaigns framed as pending plans); CLAUDE.md routing row calls `mtg-engine-type-consolidation.md`
"the active plan" while the doc self-marks COMPLETE 2026-03-09.

Root pattern: the genuinely active plans live in memory/ (`oos-retriage-plan-2026-07-18.md`,
`ef-batch-plan-2026-07-17.md`, campaign plan) while docs/ planning files fossilize without
supersession banners.

  end-of-milestone trigger; a pass is due now.
- Under the policy **as written**, only ~26 files / ~0.33MB are archivable today: 6 top-level

  says 0 HIGH — needs one disambiguating line to prevent misreading.

  slot filename.

## What is healthy
Generated reporting (`authoring-status.md` + guide + `tools/authoring-report.py`) is current
and self-dating. The SR task-record docs are exemplary (markers, snapshot-anchored). The
evergreen references (architecture, corner-cases, game-scripts, engine_explanation,
protocol-versioning) are genuinely evergreen. `workstream-state.md` is correctly rotated and
right-sized. Historical docs that self-mark (strategic-review, network-security,
type-consolidation) do it well — the problem class is specifically *unmarked* staleness.
