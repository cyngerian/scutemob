## Primitive Specification

The engine already has `EffectFilter::CreaturesYouControl`, `OtherCreaturesYouControl`, and
`OtherCreaturesYouControlWithSubtype(SubType)`. These cover "creatures you control have X"
and "other [subtype] creatures you control get +N/+N" patterns.

Card defs are blocked on several additional filter patterns:

1. **CreaturesOpponentsControl** -- "Creatures your opponents control get -2/-2" (Elesh Norn,
   Massacre Wurm, Archetype of Endurance/Imagination)
2. **CreaturesYouControlWithSubtype(SubType)** -- "Elf creatures you control get +3/+3"
   (Ezuri, activated ability -- the source IS the creature so "other" would exclude it)
3. **CreaturesYouControlWithSupertype(SuperType)** -- "Legendary creatures you control get +1/+0"
   (Rising of the Day)
4. **CreaturesYouControlWithColor(Color)** -- "Red creatures you control have first strike"
   (Bloodmark Mentor)
5. **AttackingCreaturesYouControl** -- "Attacking creatures you control have deathtouch/double
   strike" (Ohran Frostfang, Blade Historian, Stensia Masquerade)
6. **ArtifactsYouControl** -- "Artifacts you control have shroud" (Indomitable Archangel)
7. **OtherCreaturesYouControlExcludingSubtype(SubType)** -- "Other non-Human creatures you
   control get +1/+1" (Mikaeus the Unhallowed)
8. **CreaturesYouControlExcludingSubtype(SubType)** -- "Non-Human creatures you control get
   +3/+3 until end of turn" (Return of the Wildspeaker)
9. **AttackingCreaturesYouControlWithSubtype(SubType)** -- "Attacking Vampires you control have
   deathtouch" (Crossway Troublemakers)
10. **AllCreaturesWithSubtype(SubType)** -- "Dragon creatures get +1/+1" (Bladewing the Risen,
    no controller restriction)
11. **OtherCreaturesYouControlWithSubtypes(Vec<SubType>)** -- "Other Ninja and Rogue creatures
    you control get +1/+1" (Silver-Fur Master, multi-subtype OR filter)

## CR Rule Text

**CR 604.2**: Static abilities create continuous effects. These effects are active as long as
the permanent with the ability remains on the battlefield and has the ability.

**CR 611.3**: A continuous effect generated by a static ability isn't "locked in"; it applies
at any given moment to whatever its text indicates.

**CR 611.3a**: Same as above -- dynamic re-evaluation.

**CR 613.1f**: Layer 6: Ability-adding effects, keyword counters, ability-removing effects, and
effects that say an object can't have an ability are applied.

**CR 613.4c**: Layer 7c: Effects and counters that modify power and/or toughness (but don't
set power and/or toughness to a specific number or value) are applied.

/// Applies to all creature permanents controlled by opponents of the source's controller.
///
/// Used for "Creatures your opponents control get -2/-2" (Elesh Norn, Grand Cenobite).
/// Resolved dynamically at layer-application time using `effect.source`.

/// Applies to all attacking creature permanents controlled by the source's controller.
///
/// Used for "Attacking creatures you control have deathtouch" (Ohran Frostfang),
/// "Attacking creatures you control have double strike" (Blade Historian).
/// Checks `state.combat.attackers.contains_key(&object_id)` at layer-application time
/// (dynamic, CR 611.3a). See CRITICAL NOTE in Change 2.
AttackingCreaturesYouControl,

/// Applies to all artifact permanents controlled by the source's controller.
///
/// Used for "Artifacts you control have shroud" (Indomitable Archangel).
ArtifactsYouControl,

/// Applies to all creature permanents controlled by the source's controller
/// that have the specified supertype.
///
/// Used for "Legendary creatures you control get +1/+0" (Rising of the Day).
CreaturesYouControlWithSupertype(SuperType),

/// Applies to all creature permanents controlled by the source's controller
/// that have the specified color (evaluated via layer-resolved characteristics).
///
/// Used for "Red creatures you control have first strike" (Bloodmark Mentor).
CreaturesYouControlWithColor(Color),

/// Applies to all creature permanents controlled by the source's controller
/// that do NOT have the specified subtype, excluding the source.
///
/// Used for "Other non-Human creatures you control get +1/+1 and have undying"
/// (Mikaeus, the Unhallowed). Also covers "non-Elf" effects.
OtherCreaturesYouControlExcludingSubtype(SubType),

/// Applies to all creature permanents controlled by the source's controller
/// that do NOT have the specified subtype (INCLUDING the source).
///
/// Used for instant/sorcery effects: "Non-Human creatures you control get +3/+3
/// until end of turn" (Return of the Wildspeaker mode 2).
CreaturesYouControlExcludingSubtype(SubType),

/// Applies to all attacking creature permanents controlled by the source's controller
/// that have the specified subtype.
///
/// Used for "Attacking Vampires you control have deathtouch and lifelink"
/// (Crossway Troublemakers), "Attacking Elves you control have deathtouch"
/// (Elderfang Venom).
AttackingCreaturesYouControlWithSubtype(SubType),

/// Applies to all creature permanents on the battlefield (any controller) with the
/// specified subtype.

/// Used for "Dragon creatures get +1/+1 until end of turn" (Bladewing the Risen) --
/// no controller restriction, affects all players' creatures of the type.

### Change 2: Add filter matching in `layers.rs`

**File**: `crates/engine/src/rules/layers.rs`
**Action**: Add match arms in `effect_applies_to()` function (after line 670, before the closing brace)
**CR**: 604.2, 611.3a -- dynamic evaluation at layer-application time

Each new variant follows the same pattern as `CreaturesYouControl` / `OtherCreaturesYouControlWithSubtype`:
1. Check `obj_zone == ZoneId::Battlefield`
2. Check card type (Creature for creature filters, Artifact for artifact filters)
3. Check additional condition (subtype, supertype, color, attacking status)
4. Check controller match via `effect.source`
5. Check "other" exclusion if applicable

**CRITICAL NOTE on attacking status**: There is NO `is_attacking` field on `GameObject`.
Attacking creatures are tracked in `state.combat: Option<CombatState>` where
`CombatState.attackers: OrdMap<ObjectId, AttackTarget>` (defined in
`crates/engine/src/state/combat.rs`). To check if a creature is attacking:
```rust
state.combat.as_ref().map_or(false, |c| c.attackers.contains_key(&object_id))
```
The `effect_applies_to` function already receives `state: &GameState` as its first parameter,
so this is directly accessible.

Implementation notes per variant:

- **CreaturesOpponentsControl**: `obj_zone == Battlefield && Creature && obj.controller != source.controller`
- **CreaturesYouControlWithSubtype(st)**: same as `CreaturesYouControl` + `chars.subtypes.contains(st)`
- **AttackingCreaturesYouControl**: same as `CreaturesYouControl` + `state.combat.as_ref().map_or(false, |c| c.attackers.contains_key(&object_id))`
- **ArtifactsYouControl**: `obj_zone == Battlefield && Artifact && same_controller`
- **CreaturesYouControlWithSupertype(st)**: same as `CreaturesYouControl` + `chars.supertypes.contains(st)`
- **CreaturesYouControlWithColor(c)**: same as `CreaturesYouControl` + `chars.colors.contains(c)`. NOTE: must use layer-resolved colors, not base colors (the `chars` parameter already is layer-resolved at this point in the calculation).
- **OtherCreaturesYouControlExcludingSubtype(st)**: same as `OtherCreaturesYouControl` + `!chars.subtypes.contains(st)`
- **CreaturesYouControlExcludingSubtype(st)**: same as `CreaturesYouControl` + `!chars.subtypes.contains(st)`
- **AttackingCreaturesYouControlWithSubtype(st)**: same as `AttackingCreaturesYouControl` + `chars.subtypes.contains(st)`
- **AllCreaturesWithSubtype(st)**: `obj_zone == Battlefield && Creature && chars.subtypes.contains(st)` (no controller check)
- **OtherCreaturesYouControlWithSubtypes(sts)**: same as `OtherCreaturesYouControl` + `sts.iter().any(|st| chars.subtypes.contains(st))`

**IMPORTANT**: The `chars` parameter in `effect_applies_to` is the object's characteristics as computed up to the current point in layer processing. For Layer 6 (Ability) effects, colors and types have already been resolved (Layers 4-5 come before Layer 6). For Layer 7c (PtModify) effects, all layers 1-6 are resolved. So using `chars.colors`/`chars.subtypes`/`chars.supertypes` is correct.

## Risks & Edge Cases

- **Attacking status is in CombatState, NOT on GameObject**: There is no `obj.status.is_attacking`
  field. Attacking creatures are tracked in `state.combat: Option<CombatState>` where
  `CombatState.attackers: OrdMap<ObjectId, AttackTarget>` (file: `crates/engine/src/state/combat.rs`).
  The `AttackingCreaturesYouControl` filter must check:
  `state.combat.as_ref().map_or(false, |c| c.attackers.contains_key(&object_id))`.
  Outside of combat (`state.combat == None`), no creatures are attacking, so the filter
  matches nothing. This is correct behavior for static abilities like "Attacking creatures
  you control have deathtouch" -- the grant applies only during combat.

- **Opponent detection in multiplayer**: `CreaturesOpponentsControl` must check that the object's
  controller is NOT the source's controller -- it should NOT enumerate opponents explicitly.
  In a 4-player game, all three opponents' creatures are affected. Implementation: check
  `obj.controller != source.controller` (simple negation).

- **"Can't have or gain" prevention (Archetype cards)**: PB-25 adds the `RemoveKeyword` part
  but does NOT implement the "can't have or gain" prevention. This is a separate engine
  feature (keyword prevention/lock) and should be left as a TODO on those specific abilities.

- **Color filter uses layer-resolved colors**: `CreaturesYouControlWithColor(Red)` checks
  `chars.colors.contains(Red)` where `chars` is the partially-resolved characteristics at
  the current layer position. For Layer 6 (Ability) effects, colors are already resolved
  (Layer 5). This is correct per CR 613 layer ordering. But if a color-changing effect and
  a color-filtered grant are in the same layer, dependency resolution may matter. Low risk
  for current cards.

- **Dynamic LayerModification for Craterhoof/Joraga Warcaller**: These cards need
  `ModifyBoth(X)` where X is computed from game state (creature count, counter count).
  This is NOT an EffectFilter gap -- it's a `LayerModification` gap (needs dynamic amounts).
  Out of scope for PB-25.

- **Mana ability grants (Citanul Hierophants, Cryptolith Rite, etc.)**: Granting activated
  abilities (not just keywords) to creatures is a separate gap. Not in PB-25 scope.

- **Multiple statics on one card**: Several cards (Elesh Norn, Archetype of Imagination) need
  multiple `AbilityDefinition::Static` entries for different effects. This is already supported
  by the engine (vec of abilities) -- no special handling needed.

- **Vec<SubType> in OtherCreaturesYouControlWithSubtypes**: The `Vec<SubType>` payload means
  this variant is slightly different from the single-subtype variants. Hash implementation must
  iterate and hash each subtype. The `layers.rs` match uses `sts.iter().any(|st| chars.subtypes.contains(st))`
  for OR semantics.

- **Phased-out creatures**: The existing `effect_applies_to` function already excludes phased-out
  permanents for non-SingleObject filters (line 520-527 in layers.rs). The new variants inherit
  this behavior automatically since they are all battlefield-scoped filters.
