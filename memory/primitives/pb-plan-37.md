## Assessment Summary

The original G-26 estimate was ~66 cards with unique activated abilities. PB-23 through PB-36
closed the vast majority of these gaps. After systematic scanning of all remaining TODO comments
in card defs, the residual gaps that require **new engine primitives** fall into 4 small categories.

Many remaining TODOs are:
- **Backfill-only** (can use existing primitives, just need card-def wiring) -- handled in BF-1
- **Blocked on M10** (require interactive player choice: color choice, library ordering, etc.)
- **Triggered ability gaps** (not activated ability gaps -- belong to other gap groups)
- **GrantActivatedAbility** (granting activated abilities to other permanents is a major subsystem, deferred to post-alpha or M11)

Items explicitly **deferred from this batch**:
- **GrantActivatedAbility** continuous effect (~8 cards: Urza's Saga, Paradise Mantle, Bootlegger's Stash, Umbral Mantle, Thornbite Staff, Song of Freyalise, Enduring Vitality) -- too complex for 1-2 sessions; requires a new ContinuousEffect that registers activated abilities on other permanents. Deferred to post-alpha.
- **"Activate as though haste" static** (2 cards: Tyvar Jubilant Brawler, Thousand-Year Elixir) -- requires a game-restriction-level override of summoning sickness for activations. Small but niche. Deferred.
- **Effect::ChangeTarget** (3 cards: Bolt Bend, Hydroelectric Specimen, Untimely Malfunction) -- changing a spell's target on the stack requires stack-manipulation infrastructure; blocked on M10.
- **Color choice at activation** (Mother of Runes, Alseid, Commander's Plate, Nykthos, etc.) -- blocked on M10 interactive player choice.
- **Level Up mechanic** (Joraga Treespeaker, Class cards) -- entire subsystem, not a single primitive.
- **Cost::ReturnPermanentToHand** (3 cards: Scryb Ranger, Meloku, Mina and Denn) -- bounce-as-cost needs a new Cost variant + harness wiring + legal action filtering; achievable but only 3 cards benefit. Include only if session time permits.

## CR Rule Text

### CR 603.4 -- Intervening-If
> 603.4 A triggered ability may read "When/Whenever/At [trigger event], if [condition], [effect]."
> ... The ability triggers only if the condition is true at the time the trigger event occurs.
> The ability checks that condition again when it resolves. If the condition is false at either
> of those times, the ability does nothing.

Relevant to Condition::WasCast: "When [this] enters, **if you cast it**, [effect]."
The "if you cast it" is an intervening-if condition that checks whether the permanent
entered via casting (from the stack) rather than being put directly onto the battlefield
(flicker, reanimate, Aether Vial, etc.).

### CR 611.2b -- Duration
> 611.2b A continuous effect generated by the resolution of a spell or ability lasts as
> long as stated by the spell or ability creating it (such as "until end of turn"). If no
> duration is stated, it lasts until end of game.

"Until your next turn" is a stated duration distinct from UntilEndOfTurn (CR 514.2).
It persists through opponents' turns and expires at the start of the controller's
next turn (beginning of upkeep, before any triggers).

### CR 602.5b -- Activation Restrictions
> 602.5b A player can't begin to activate an activated ability that has a restriction on
> its use unless the restriction is met.

"Activate only once each turn" is an activation restriction (CR 602.5g).

### CR 602.2 -- Activation Costs
> 602.2 To activate an ability, a player first puts it on the stack, then pays the costs.

"Tap an untapped creature you control" is a cost (CR 602.2) -- the tapping happens
as part of cost payment, before the ability goes on the stack for resolution.

### Change 10: Track activations per turn for once-per-turn enforcement

**File**: `crates/engine/src/state/game_object.rs`
**Action**: Add `#[serde(default)] pub abilities_activated_this_turn: u32` to `GameObject`

This counter is incremented when an activated ability from this object goes on the stack.
It is reset to 0 at the beginning of each turn (in the untap step sweep). The
`once_per_turn` check compares this counter against 0 at activation time.

Note: This is a simplified approach. Cards with multiple activated abilities where only one
is once-per-turn would need per-ability tracking. For the 3 cards in scope (Ramos, Steel
Hellkite, Scryb Ranger), all have exactly one once-per-turn ability, so a per-object counter
suffices. A more precise approach (per-ability-index tracking) is deferred.

`once_per_turn == true` and the source object's `abilities_activated_this_turn > 0`.

## Risks & Edge Cases

- **was_cast propagation through copy effects**: If a copy of a permanent enters, it was not cast. The `was_cast` field should NOT be copied in Layer 1 copy effects. Verify `copy.rs` does not copy this field.
- **was_cast and morph/mutate**: Face-down cards cast via morph are still "cast." Mutate targets that merge are "cast." These should have `was_cast = true`. Verify resolution paths for morph/mutate set the field.
- **UntilYourNextTurn in extra turns**: If a player takes an extra turn, "until your next turn" expires at the start of the extra turn (it IS the next turn). This is correct behavior per CR.
- **Once-per-turn with multiple activated abilities**: The simplified per-object counter works for Ramos, Steel Hellkite, and Scryb Ranger (each has exactly one once-per-turn ability). If future cards have multiple activated abilities where only some are once-per-turn, the tracking needs to be per-ability-index. Document this limitation.
- **UntilYourNextTurn and conceding players**: If the player whose "next turn" we're waiting for concedes, the effect should expire immediately (their turn will never come). Add a check in the expiry logic or SBA.
- **GrantPlayerProtection duration**: Currently GrantPlayerProtection directly modifies PlayerState. To support UntilYourNextTurn duration, it may need to be reworked to use ApplyContinuousEffect with a special LayerModification, or the GrantPlayerProtection effect itself needs a duration field. Evaluate at implementation time which approach is simpler.
